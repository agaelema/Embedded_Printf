    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-embedded_printf" class="anchor" href="#embedded_printf" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" role="img" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Embedded_Printf</h1>

<p>Embedded version of the famous "printf( )" function.</p>

<blockquote>
<p>This code is based in the code shared in this link: <a href="http://e2e.ti.com/support/development_tools/code_composer_studio/f/81/p/30479/107146">http://e2e.ti.com/support/development_tools/code_composer_studio/f/81/p/30479/107146</a></p>
</blockquote>

<p>The idea is create an simple and efficient library to meet some common needs in embedded systems, as:</p>

<ul>
<li>convert numbers to strings</li>
<li>send strings over serial port</li>
<li>convert a string into number</li>
</ul>

<p>To maintain the ballance between code size, speed and functionalities it's possible to do little changes in the code using C preprocessor.</p>

<ul>
<li>enable padding</li>
<li>enable float number</li>
<li>increase precision (using double)</li>
<li>enable extra decimal places</li>
<li>enable binary notation</li>
</ul>

<p>All of this commenting/uncommenting the preprocessors.</p>

<p>The famous "printf( )" functions is named as "embedded_printf( )" and use a similar sintax as the original function with some reductions.</p>

<p>Besides the "embedded_printf( )" there are a simple version of each principal function. This can be useful when the traditional sintax is not required or when the speed of code/size is more important.</p>

<blockquote>
<p>To facilitate the portability the code is modular and the configuration of the serial port is done in separate files ("serial_conf.h" and "serial_conf.c").</p>

<p>If possible contribute sending bug repports, new functions or porting the library to new devices.</p>
</blockquote>

<p>In this initial version is provided the port to the MSP430G2553 used in the Launchpad Valueline.</p>
